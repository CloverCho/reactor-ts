Action

An action denotes a self-scheduled event. If an action is instantiated without a delay, then the time interval between the moment of scheduling this action (cause), and a resulting reaction (effect) will be determined upon the call to schedule. If a delay is specified, it is considered constant and cannot be overridden using the delay argument in a call to schedule().

Reactor

Each component has a name. It will typically also acquire a parent, unless it is a top-level composite. The parent property is set/unset once a component is added to or removed from a container. Adding a component to a container will also ensure that it is uniquely indexed within that container.
